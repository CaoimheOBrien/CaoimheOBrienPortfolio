Behind This Project

I created my first portfolio in Semester 2 in Year 2 of my degree and I was still getting the "hang" of UI design and user-centered ideologies.

Since then, I have learned a lot more about web design and development and I wanted to create a new portfolio that better reflects my skills and personality.

This portfolio was created using HTML, CSS, and JavaScript.

I started by sketching out a basic layout and design for the website, focusing on simplicity and ease of navigation. I used criticisms that I recieved on my previous portfolio to improve the user experience on my old portfolio in my time as a Summer intern at UNUM, as noted in the image below.

I used Figma to create a more detailed design and layout for the website; including color schemes, typography, and imagery. I wanted to make sure that the website was visually appealing and easy to read. I also implemented variables and other design systems in my process.

Using my Low and Mid Fidelity designs, I created a prototype that I could test and refine. I made sure to get feedback from friends and family to ensure that the website was user-friendly and easy to navigate.

Once I had a solid design, I began coding the website using HTML, CSS, and JavaScript on Visual Studio Code. Originally, I had planned to repurpose my old site and just updated teh styles but it ended up being more work than I had anticipated so I decided to start from scratch.

I used grays and blues to give the website a professional and modern look.

Overall, I am very happy with how this portfolio turned out. It showcases my skills and personality in a way that I am proud of, and I believe it will help me stand out.

Behind This Project

Here is the link to the live page for this website: Sage Academy Live link

Here is the link to the GitHub repo: GitHub Link

This website was created as a group project for my Web UI Design module in Semester 2 of Year 2 of my degree.

Our group was tasked with creating a website for a fictional educational institution called "Sage Academy". The website was to be designed to attract potential students and provide information about the courses and programs offered by the academy.

As a group, we started by researching other educational websites to get an idea of what worked well and what didn't. We also brainstormed ideas for the layout and design of the website.

Once we had a solid plan in place, we divided the work among the group members.

I moved towards a leadership role in this project, as I had more experience with web design and development than some of the other group members. I took on the role of project manager, ensuring that everyone was on track and that the project was progressing smoothly.

We used PenPot to create wireframes and prototypes for the website. This allowed us to visualize the layout and design of the website before we started coding.

I was tasked with creating the sign-in, contact us, and quizzes pages for the website. I also created several components that were used throughout the site, such as buttons, cards and forms.

We used a combination of HTML, CSS, and JavaScript to create the website. We also made sure to optimize the website for mobile devices, as we knew that many potential students would be accessing the site from their phones.

Overall, I am very proud of the work that our group did on this project. The website turned out great, and Ilearned a lot about working through conflict resolution and collaboration in a team setting.

I created this portfolio website as part of my Web UI a Design module in Semester 2 of Year 2 of my degree.

The website was designed to showcase my skills and experience as a web designer and developer. I wanted to create a website that was visually appealing, easy to navigate, and provided a sense of who I was not only as a designer but also as a creative.

I started out by looking at other portfolio websites for inspiration. I took note of the layouts, color schemes, and typography that I liked and used them to inform my own design. I liked the use of dark colours in a lot of the sites but when I implemented into my own site, I don't think it came across as well.

I used PenPot to create wireframes and prototypes for the website. This allowed me to visualize the layout and design of the website before I started coding.

I used a combination of HTML, CSS, and JavaScript to create the website. I made sure to optimize the website for mobile devices, as I knew that many potential employers would be accessing the site from their phones.

When I first finnished this project I was very happy with its outcome but now I feel that it does not accurately represent my skills and personality as a designer. I have learned a lot since creating this website, so I created a new portfolio that better reflects who I am and what I can do under the guidance of my internship at UNUM.

My Old Portfolio

I created this 2D animation as part of a Concept Art for Games module in Semester 1 of Year 2 of my degree.

The animation was created using a pixelated art style to give it a retro feel. I used a limited color palette to create a cohesive look and feel for the animation.

I started by sketching out the basic design for the enemy character. I then created a series of frames for the animation using a software called Aesprite, focusing on the movement and actions of the character.

I used a combination of traditional animation techniques and digital tools to create the animation. I made sure to pay attention to the timing and spacing of the animation to ensure that it was smooth and fluid, especially since it was going ot be used in a game.

Overall, I am very happy with how this animation turned out. It showcases my skills as an animator and my ability to create engaging and visually appealing content.

My Old Portfolio

I created this 2D game as part of a UI prototyping module in Semester 1 of Year 2 of my degree. Our prompt was to make a game based on the theme of "Utopia" and I felt that my perfect utopia was a nice cozy coffee shop.

I first created all the assets for this game in Aesprite, using a pixelated art style to give it a retro feel. I used a limited color palette to create a cohesive look and feel for the game as I watned it to have a "cozy" vibe.

I then used HTML, CSS, and JavaScript to create the game. This was the first time I had used the canvas function in HTML so it was really fun to explore this because it was like I was coding in C++ which I hadd done in my first year of university.

I made sure to pay attention to the user experience and interface design to ensure that the game was intuitive; just in case a user got confused there is a list of instructions in the top left corner of the game screen.

Unlike my other projects, this was hosted on codeberg which is an alternative to GitHub, however I prefer GitHub as it is easier to launch pages and more widely used.

Overall, I am very happy with how this game turned out. It showcases my skills as a game designer and my ability to create engaging and visually appealing content.

My Old Portfolio

This is a poster that I worked on in my personal time. I worked with an optometrist to create a poster to highlight certain statistics about eye health that they had recently written a paper on. He wanted an aide for his presentation to the Eurpopean Union of Optometry.

I used Adobe Photoshop to create the poster, focusing on a clean and professional design. I used a combination of images and text to create a visually appealing poster that would catch the attention of potential patients.

I made sure to pay attention to the typography and layout of the poster to ensure that it was easy to read and understand.

I used a color scheme that was consistent with the optometrist's branding, using blues and greens to create a calming and professional look as well as ensuring that it could be read by people with a variety of different levels of colour blindess.

Overall, I am very happy with how this poster turned out. It showcases my skills as a graphic designer and my ability to create engaging and visually appealing content.
